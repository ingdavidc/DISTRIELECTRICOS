# System Rules for this Project

- Cuando se apliquen cambios y actualizaciones (por ejemplo, al cambiar IDs, nombres de categorías, variables de entorno, etc.), el agente DEBE realizar una verificación profunda, exhaustiva y profesional de toda la integración. Esto incluye revisar módulos, páginas principales, componentes de UI, el panel de administración, y verificar que no queden referencias rotas. Siempre buscar con `findstr` o el buscador disponible el UUID o string viejo antes de dar por terminado un trabajo de refactorización o eliminación.
